Pakiet w języku Java jest mechanizmem służącym do organizowania klas, interfejsów, rekordów, enumów, adnotacji oraz innych elementów programu.

Pakiety pełnią podobną rolę jak foldery w systemie operacyjnym.

Ich głównym zadaniem jest porządkowanie kodu źródłowego.

Pakiety ułatwiają zarządzanie dużymi projektami.

Pakiety zmniejszają ryzyko konfliktów nazw pomiędzy klasami.

Pakiety umożliwiają logiczny podział aplikacji.

Pakiety zwiększają czytelność kodu.

Pakiety poprawiają jego utrzymywalność.

Każdy plik Java może należeć tylko do jednego pakietu.

Deklaracja pakietu znajduje się zawsze na początku pliku źródłowego.

Deklaracja pakietu poprzedza wszystkie importy.

Przykład deklaracji pakietu wygląda następująco.

```java
package com.company.project.model;
```

Jeżeli plik nie zawiera deklaracji package, należy do pakietu domyślnego.

Pakiet domyślny nazywany jest Default Package.

Nie zaleca się używania pakietu domyślnego w profesjonalnych projektach.

Pakiety tworzą hierarchię.

Hierarchia zapisywana jest przy użyciu kropek.

Każda kropka odpowiada kolejnemu katalogowi.

Przykład.

```text
com.company.project.model
```

Odpowiada strukturze katalogów.

```text
com
 └── company
      └── project
           └── model
```

Nazwy pakietów zgodnie z konwencją zapisuje się małymi literami.

Nie stosuje się wielkich liter.

Nie stosuje się spacji.

Nie stosuje się znaków specjalnych.

Najczęściej wykorzystuje się odwrotną nazwę domeny internetowej.

Przykład.

```text
pl.pjatk.student.project
```

lub

```text
com.google.gson
```

Takie nazewnictwo gwarantuje unikalność pakietów.

Pakiety mogą zawierać klasy.

Pakiety mogą zawierać interfejsy.

Pakiety mogą zawierać rekordy.

Pakiety mogą zawierać enumy.

Pakiety mogą zawierać adnotacje.

Pakiety mogą zawierać podpakiety.

Podpakiety są niezależnymi pakietami.

Pakiet parent nie ma automatycznego dostępu do elementów podpakietu.

Podpakiet również nie ma automatycznego dostępu do pakietu nadrzędnego.

Przykład.

```text
com.company
com.company.model
```

Są to dwa różne pakiety.

Pakiety wykorzystują mechanizm importowania klas.

Import umożliwia korzystanie z klas znajdujących się w innych pakietach.

Przykład.

```java
import java.util.ArrayList;
```

Możliwe jest również importowanie wszystkich klas z danego pakietu.

```java
import java.util.*;
```

Import z gwiazdką nie importuje podpakietów.

Importowane są jedynie klasy znajdujące się bezpośrednio w danym pakiecie.

Import nie powoduje kopiowania klasy.

Import jedynie informuje kompilator, gdzie znajduje się dana klasa.

Możliwe jest korzystanie z pełnej nazwy klasy.

Przykład.

```java
java.util.ArrayList<String> list = new java.util.ArrayList<>();
```

W takim przypadku import nie jest potrzebny.

Najważniejszym pakietem języka Java jest java.lang.

Pakiet java.lang importowany jest automatycznie.

Nie trzeba go importować ręcznie.

Pakiety posiadają własny poziom dostępu.

Jeżeli przed klasą lub metodą nie podano żadnego modyfikatora dostępu, obowiązuje dostęp package-private.

Element package-private jest dostępny wyłącznie wewnątrz tego samego pakietu.

Przykład.

```java
class Person {
}
```

Klasa Person będzie widoczna jedynie w swoim pakiecie.

Pakiety są podstawą organizacji projektów Maven.

Struktura projektu najczęściej wygląda następująco.

```text
src
 └── main
      └── java
           └── com
                └── company
                     └── project
                          ├── model
                          ├── service
                          ├── repository
                          ├── controller
                          ├── util
                          ├── config
                          ├── exception
                          └── Main.java
```

Pakiety umożliwiają również grupowanie klas według wzorców projektowych.

Każdy wzorzec może posiadać własny pakiet.

Pakiety współpracują z modułami wprowadzonymi w Java 9.

Moduł może zawierać wiele pakietów.

Moduł decyduje, które pakiety będą dostępne dla innych modułów.

Informacje o modułach znajdują się w pliku `module-info.java`.

Pakiety są jednym z podstawowych mechanizmów organizacji kodu w języku Java.

Bez odpowiednio zaprojektowanej struktury pakietów utrzymanie dużych projektów staje się bardzo trudne.

Znajomość pakietów jest niezbędna podczas pracy z Maven, Gradle, Spring Boot, Jakarta EE, bibliotekami zewnętrznymi oraz wszystkimi nowoczesnymi projektami tworzonymi w języku Java.
